# 进度总结 – SyncTube 项目 (2026-07-02)

## 1. 项目概述
**目标**：构建一个实时协作视频观看 Web 应用（"同步影院"），支持多用户在同一房间内同步播放/暂停/进度，并附带文字聊天、在线用户列表。

**技术栈**：Next.js 15 (App Router)、React 19、TypeScript、Tailwind v3、shadcn/ui 风格原语、Framer Motion、Phosphor 图标、`react-player`、`qrcode.react`、B站扫码登录 + 服务端流代理（1080P）、Bilibili iframe 降级、mock Socket.IO 层（BroadcastChannel）。

**设计基调**：暗色玻璃拟态（dark glassmorphism），单一霓虹蓝强调色 `#00d4ff`，圆角系统统一（`xl` / `2xl` / `full`），无 em dash，零 AI 味。

## 3. 关键功能实现细节

### 视频同步机制
- 使用 `react-player` 支持 MP4、HLS、YouTube；B站链接在登录后通过服务端流代理用 `react-player` 播放（支持同步控制），未登录时降级为 iframe 嵌入（无法精确同步）。
- `useVideoSync` hook 管理播放状态，通过 `applyingRemote` ref 防止循环更新。
- `currentTime` 同时维护 ref（回调访问）和 state（UI 响应），进度条与时间显示实时更新。
- 新用户加入时自动同步当前视频 URL、播放状态和进度（`room:sync` 事件）。
- 自定义进度条支持拖拽 seek，拖拽时通过 `video:seek` 事件广播给房间内所有用户。

### B站 1080P 扫码登录与直链播放
- **服务端 QR 登录**：调用 `passport.bilibili.com` 的 Web QR 登录 API，用户用B站 App 扫码确认后，服务端捕获 `Set-Cookie` 中的 `SESSDATA`、`bili_jct`、`DedeUserID` 等 cookie，存入内存 session（`Map<sessionId, BilibiliCookies>`），通过 HTTP-only Cookie `synctube_session` 关联用户。
- **视频解析**：登录用户请求 `qn=80`（1080P），未登录用户请求 `qn=32`（480P）。先通过 `api.bilibili.com/x/web-interface/view` 获取视频 `cid`，再通过 `api.bilibili.com/x/player/playurl` 获取 CDN 直链。
- **流代理**：B站 CDN 要求 `Referer: https://www.bilibili.com` 头部，浏览器无法跨域设置，因此通过 `/api/bilibili/stream` 服务端代理。代理支持 HTTP `Range` 请求，使 `react-player` 可以拖拽 seek。
- **前端集成**：`page.tsx` 检测到B站 URL 后调用 `/api/bilibili/resolve` 获取流地址，传给 `VideoPlayer` 的 `streamUrl` prop，`react-player` 直接播放代理流而非 iframe。登录状态通过 `BilibiliLogin` 组件（模态弹窗 + `QRCodeSVG`）触发，扫码成功后自动重新解析获取 1080P 流。

### 聊天与用户列表
- 文字聊天通过 `useChat` 管理，消息上限 200 条，支持滚动自动到底部（`ScrollArea` ref）。
- 用户列表实时更新（加入/离开），昵称在加入前通过 `NicknameGate` 设定，避免幽灵用户。
- 用户离开房间时 `useRoom` 的 cleanup 调用 `socket.disconnect()`，触发 `leaveRoom` 广播 `user:left` 事件，其他用户实时感知。

### 状态同步层（Mock Socket）
- `MockSocket` 类实现 `socket.io-client` 接口（`on`/`off`/`emit`/`disconnect`）。
- 使用 `BroadcastChannel` 在同一浏览器不同标签页间广播事件，实现多窗口同步测试。
- 后续只需将 `getSocket()` 替换为 `io(URL)` 即可无缝迁移到真实 Socket.IO 服务器。

### 设计一致性
- **玻璃拟态**：`backdrop-blur-sm`、半透明深色背景、细边框 `border-white/5`。
- **强调色**：所有交互元素（按钮、链接、焦点状态）统一使用 `#00d4ff`（`bg-blue-400` / `text-blue-400`）。
- **圆角规范**：按钮/输入框 `rounded-xl`，卡片/面板 `rounded-2xl`，头像/徽章 `rounded-full`。
- **动画**：Framer Motion 入场动画（主页标题、房间加载）、微交互反馈（悬停、点击）。
- **无障碍**：尊重 `prefers-reduced-motion` 和 `prefers-reduced-transparency`。

## 5. 已实现的用户流程

1. **进入主页** → 动画标题 + "创建房间" / "加入房间" 按钮。
2. **创建房间** → 生成随机 6 位房间码，跳转到 `/room/[id]`。
3. **加入房间** → 输入房间码，点击加入，跳转。
4. **房间内**：
   - 弹出昵称门控 → 输入昵称（必填）。
   - 主界面：左侧视频播放器 + 控制条，右侧聊天面板 + 用户列表。
   - 粘贴视频 URL → 自动加载。B站链接会自动调用 `/api/bilibili/resolve` 解析：未登录时降级为 iframe（仅 480P，无同步），登录后通过流代理用 `react-player` 播放（1080P，支持同步控制）。其他格式直接用 `react-player`。
   - B站未登录时控制条显示"扫码登录"按钮和提示文案；登录后显示"B站已登录 1080P"徽章，播放/暂停/进度条生效。
   - 播放/暂停同步到所有用户；自定义进度条支持拖拽 seek 并广播。
   - 聊天消息实时广播。
   - 用户列表显示所有在线成员（含自身）。
5. **离开房间** → 关闭标签页或点击返回主页，`useRoom` cleanup 调用 `socket.disconnect()`，广播 `user:left`，其他用户实时感知。

## 9. B站 1080P 扫码登录实现明细（2026-07-02）

### 架构决策
采用**服务端 QR 登录 + 直链流代理**方案。用户在 SyncTube 前端扫码，服务端代为完成B站登录并持有 cookie，后续用该 cookie 请求 1080P CDN 直链，再通过服务端代理转发给前端 `react-player` 播放。这样解决了浏览器无法设置 `Referer` 跨域头的问题，同时让B站视频从 iframe 降级方案升级为完全可控的 `react-player` 播放，实现了播放/暂停/进度的精确同步。

### 新增文件

#### `lib/bilibili.ts` — B站 API 集成核心
- **Session 管理**：`SESSION_COOKIE = "synctube_session"`，`cookieStore = new Map<string, BilibiliCookies>()`。通过 `next/headers` 的 `cookies()` 读写 HTTP-only Cookie 关联用户。
- **`generateQrCode()`**：调用 `passport.bilibili.com/x/passport-login/web/qrcode/generate`，返回 `{ qrUrl, qrcodeKey, sessionId }`。
- **`pollQrLogin(qrcodeKey, sessionId)`**：轮询 `passport.bilibili.com/x/passport-login/web/qrcode/poll`，状态码 0=等待、86038=已扫码未确认、0+Set-Cookie=成功、86090=二维码过期。成功时从 `Set-Cookie` 头部提取 `SESSDATA`、`bili_jct`、`DedeUserID` 等 cookie 存入 session。
- **`resolveBilibiliVideo(bvid, sessionId)`**：先调 `api.bilibili.com/x/web-interface/view` 获取 `cid`，再调 `api.bilibili.com/x/player/playurl` 获取流地址。登录用户 `qn=80`（1080P），未登录 `qn=32`（480P）。返回 `{ streamUrl, title, cover, duration, quality, loggedIn }`。
- **`proxyBilibiliStream(targetUrl, sessionId, range)`**：带 `Referer` 和 cookie 头部请求B站 CDN，转发 `Response` 给前端，支持 `Range` 请求实现拖拽 seek。
- **`extractBvId(url)`**：从各种B站 URL 格式中解析 `BV` ID。

#### API 路由（4 个）
- **`/api/bilibili/qrcode`** (`app/api/bilibili/qrcode/route.ts`)：`GET` 调用 `generateQrCode()`，返回 `{ qrUrl, qrcodeKey, sessionId }`。
- **`/api/bilibili/poll`** (`app/api/bilibili/poll/route.ts`)：`GET` 接收 `qrcodeKey` 参数，调用 `pollQrLogin()`，返回 `{ status: "waiting"|"scanned"|"success"|"expired" }`。
- **`/api/bilibili/resolve`** (`app/api/bilibili/resolve/route.ts`)：`GET` 接收 `url` 参数，提取 BV ID 后调用 `resolveBilibiliVideo()`，返回 `{ streamUrl: "/api/bilibili/stream?u=...", title, cover, duration, quality, loggedIn, bvid }`。
- **`/api/bilibili/stream`** (`app/api/bilibili/stream/route.ts`)：`GET` 代理B站 CDN 视频流，支持 `Range` 请求头实现拖拽进度。

#### `components/room/BilibiliLogin.tsx` — 扫码登录 UI
- 使用 `qrcode.react` 的 `QRCodeSVG` 渲染二维码。
- 状态机：`idle` → `loading` → `waiting` → `scanned` → `success`（或 `expired`/`error`）。
- 每 2 秒轮询 `/api/bilibili/poll`，成功后调用 `onLogin` 回调触发重新解析。
- 毛玻璃模态弹窗，Framer Motion 入场动画。

### 用户体验流程
1. 用户粘贴B站视频链接 → 前端自动调 `/api/bilibili/resolve` 解析。
2. 未登录 → 返回 480P 流地址但前端仍用 iframe 播放（降级），控制条显示"扫码登录"按钮。
3. 用户点击扫码 → 弹出模态二维码 → 用B站 App 扫码确认。
4. 轮询检测到登录成功 → 关闭弹窗，自动重新解析获取 1080P 流地址。
5. `VideoPlayer` 切换到 `react-player` 播放代理流，播放/暂停/进度条全部生效并同步给房间内其他用户。
