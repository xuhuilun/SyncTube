# 进度总结 – SyncTube 项目 (2026-07-02)

## 1. 项目概述
**目标**：构建一个实时协作视频观看 Web 应用（"同步影院"），支持多用户在同一房间内同步播放/暂停/进度，并附带文字聊天、在线用户列表。

**技术栈**：Next.js 15 (App Router)、React 19、TypeScript、Tailwind v3、shadcn/ui 风格原语、Framer Motion、Phosphor 图标、`react-player`、Bilibili iframe 降级、mock Socket.IO 层（BroadcastChannel）。

**设计基调**：暗色玻璃拟态（dark glassmorphism），单一霓虹蓝强调色 `#00d4ff`，圆角系统统一（`xl` / `2xl` / `full`），无 em dash，零 AI 味。

## 3. 关键功能实现细节

### 视频同步机制
- 使用 `react-player` 支持 MP4、HLS、YouTube；Bilibili 链接自动转 iframe 嵌入（通过 `lib/video.ts` 识别并生成 `player.html` 嵌入 URL）。
- `useVideoSync` hook 管理播放状态，通过 `applyingRemote` ref 防止循环更新。
- `currentTime` 同时维护 ref（回调访问）和 state（UI 响应），进度条与时间显示实时更新。
- 新用户加入时自动同步当前视频 URL、播放状态和进度（`room:sync` 事件）。
- 自定义进度条支持拖拽 seek，拖拽时通过 `video:seek` 事件广播给房间内所有用户。

### 聊天与用户列表
- 文字聊天通过 `useChat` 管理，消息上限 200 条，支持滚动自动到底部（`ScrollArea` ref）。
- 用户列表实时更新（加入/离开），昵称在加入前通过 `NicknameGate` 设定，避免幽灵用户。
- 用户离开房间时 `useRoom` 的 cleanup 调用 `socket.disconnect()`，触发 `leaveRoom` 广播 `user:left` 事件，其他用户实时感知。

### 状态同步层（Mock Socket）
- `MockSocket` 类实现 `socket.io-client` 接口（`on`/`off`/`emit`/`disconnect`）。
- 使用 `BroadcastChannel` 在同一浏览器不同标签页间广播事件，实现多窗口同步测试。
- 后续只需将 `getSocket()` 替换为 `io(URL)` 即可无缝迁移到真实 Socket.IO 服务器。

### 设计一致性
- **玻璃拟态**：`backdrop-blur-sm`、半透明深色背景、细边框 `border-white/5`。
- **强调色**：所有交互元素（按钮、链接、焦点状态）统一使用 `#00d4ff`（`bg-blue-400` / `text-blue-400`）。
- **圆角规范**：按钮/输入框 `rounded-xl`，卡片/面板 `rounded-2xl`，头像/徽章 `rounded-full`。
- **动画**：Framer Motion 入场动画（主页标题、房间加载）、微交互反馈（悬停、点击）。
- **无障碍**：尊重 `prefers-reduced-motion` 和 `prefers-reduced-transparency`。

## 5. 已实现的用户流程

1. **进入主页** → 动画标题 + "创建房间" / "加入房间" 按钮。
2. **创建房间** → 生成随机 6 位房间码，跳转到 `/room/[id]`。
3. **加入房间** → 输入房间码，点击加入，跳转。
4. **房间内**：
   - 弹出昵称门控 → 输入昵称（必填）。
   - 主界面：左侧视频播放器 + 控制条，右侧聊天面板 + 用户列表。
   - 粘贴视频 URL → 自动加载（B站 → iframe，其他 → react-player）。
   - 播放/暂停同步到所有用户；自定义进度条支持拖拽 seek 并广播。
   - 聊天消息实时广播。
   - 用户列表显示所有在线成员（含自身）。
5. **离开房间** → 关闭标签页或点击返回主页，`useRoom` cleanup 调用 `socket.disconnect()`，广播 `user:left`，其他用户实时感知。

## 8. Bug 修复与代码清理明细（2026-07-02）

### 编译修复
- **`tsconfig.json`**：将 `server/` 和 `Skill/` 排除出编译范围（`server/index.ts` 引用未安装的 `socket.io`，`Skill/` 含无关示例文件）。
- **`React.MutableRefObject` 弃用**：在 `VideoPlayer.tsx` 和 `useVideoSync.ts` 中替换为 `React.RefObject`（React 19 已将 `RefObject.current` 改为可写，弃用 `MutableRefObject`）。
- **`video:state` 事件类型不匹配**：`ClientToServerEvents` 使用 `VideoState` 而 `ServerToClientEvents` 使用 `VideoStatePayload`（`{ videoState: VideoState }`），交集类型导致 `socket.on` 类型报错。统一为 `VideoStatePayload`，更新 `useVideoSync.ts` emit 调用包裹 payload，`mockSocket.ts` emit 处理器解包 payload。

### 运行时 Bug 修复
- **`useRoom` cleanup 不断开连接**：用户离开房间时只移除事件监听器，未调用 `socket.disconnect()`，导致用户永久在线。修复：在 cleanup 中添加 `socket.disconnect()`，触发 `leaveRoom` 广播 `user:left`。
- **`VideoControls` 忽略 `onSeek` prop**：`onSeek` 在接口中声明但从未解构或使用，自定义进度条功能无效。修复：解构 `onSeek` 和新增 `duration` prop，添加 `<input type="range">` 进度条，`onChange` 调用 `onSeek` 实现拖拽 seek 并广播。
- **`useVideoSync` 的 `currentTime` 不响应**：仅用 ref 存储不触发重渲染，时间显示和进度条停滞。修复：添加 `currentTime` state 镜像 ref，在所有赋值点同步更新，返回 state 而非 ref。

### 其他修复
- **`b23.tv` 短链接内嵌**：`getBilibiliEmbed` 原将 `b23.tv` 原始 URL 作为 iframe src，但 `b23.tv` 重定向到 `bilibili.com`（非 `player.bilibili.com`），被 X-Frame-Options 阻止。修复：返回 `null`，`VideoControls` 显示专门提示引导用户使用完整链接。

### 死代码清理
- 移除 `hooks/useSocket.ts`（从未被任何文件导入）。
- 移除空目录 `app/api/`（`app/api/rooms/route/` 为空，无 `route.ts`）。
- 从 `package.json` 移除 `uuid`、`@types/uuid`、`dayjs`（均未被导入，项目使用自有 `genId()` 和原生 `Date`）。
