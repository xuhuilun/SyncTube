# SyncTube Ubuntu ECS 部署方案

> 方案：直接 Node.js 运行 + PM2 守护，通过 `http://ECS公网IP:3000` 访问。
> 适用：Ubuntu 20.04 / 22.04 / 24.04 LTS，阿里云/腾讯云/华为云等 ECS 通用。

---

## 整体架构

```
浏览器  ──HTTP/WS──>  :3000  ──>  server.mjs (Node.js)
                                  ├─ Next.js (页面/API/静态资源)
                                  └─ Socket.IO (实时同步/聊天)
                                       └─ 房间状态：内存 Map（单进程）
```

要点：

- 前端和 Socket.IO 跑在**同一个端口**（3000），同一个 Node 进程。无需 nginx 反代、无需 CORS。
- 房间数据存内存 → **只能单进程**。PM2 配置已是 `exec_mode: fork, instances: 1`，不要改成 cluster。
- 无数据库、无 Redis 依赖（项目根目录 `Redis.md` 的连接信息当前代码未使用）。

---

## 0. 准备：ECS 安全组 / 防火墙

在云控制台的安全组里**入方向**放行 TCP 端口：

| 端口 | 用途 | 备注 |
|---|---|---|
| 22 | SSH | 默认已开 |
| 3000 | SyncTube Web + WebSocket | **必须放行**，否则浏览器访问不通 |

如果用 `ufw`，也要放行：

```bash
sudo ufw allow 22/tcp
sudo ufw allow 3000/tcp
sudo ufw enable        # 若尚未启用
sudo ufw status
```

## 1. 服务器环境准备（SSH 登录后执行）

### 1.1 系统更新 + 基础工具

```bash
sudo apt update && sudo apt upgrade -y
sudo apt install -y curl git build-essential
```

### 1.2 安装 Node.js 20 LTS

SyncTube 用的是 Next.js 15 + React 19，需要 Node ≥ 18.18，推荐 20 LTS。

```bash
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs

# 验证
node -v   # 应输出 v20.x.x
npm -v
```

### 1.3 安装 PM2

```bash
sudo npm install -g pm2
pm2 -v
```

### 1.4（可选）安装 git，便于以后拉代码

```bash
sudo apt install -y git
```

## 2. 上传代码到服务器

任选一种方式。

### 方式 A：用 scp 直接上传（推荐首次部署）

在**本地机器**（Windows PowerShell / Git Bash）执行：

```bash
# 把整个项目打包（排除 node_modules / .next / .git）
cd D:\Github\SyncTube
tar --exclude=node_modules --exclude=.next --exclude=.git -czf synctube.tar.gz .

# 上传到 ECS（替换为你的用户名和公网 IP）
scp synctube.tar.gz root@你的ECS公网IP:/root/

# SSH 登录服务器
ssh root@你的ECS公网IP
```

在**服务器**上：

```bash
mkdir -p /opt/synctube
tar -xzf /root/synctube.tar.gz -C /opt/synctube
cd /opt/synctube
ls -la   # 应能看到 server.mjs、package.json、app/ 等
```

### 方式 B：从 GitHub 拉取

```bash
cd /opt
sudo git clone https://github.com/你的用户名/SyncTube.git synctube
cd synctube
```

> 注意：`node_modules`、`.next` 不应上传，服务器上要重新装、重新 build。

## 3. 安装依赖 + 构建生产产物

```bash
cd /opt/synctube

# 安装依赖
npm install --omit=dev   # 或 npm ci --omit=dev（更严格，需 package-lock.json）

# 构建生产产物（必须！server.mjs 生产模式会读 .next/）
npm run build
```

`npm run build` 完成后会生成 `.next/` 目录，`server.mjs` 在 `NODE_ENV=production` 下用它来处理请求。

构建成功的标志：终端看到 `✓ Compiled successfully` 且 `.next/BUILD_ID` 文件存在：

```bash
ls .next/BUILD_ID   # 应输出一个文件名
```

---

## 4. 用 PM2 启动 + 守护

### 4.1 启动

```bash
cd /opt/synctube
pm2 start ecosystem.config.cjs
pm2 status
```

应看到 `synctube` 进程 status 为 `online`，restart 数为 0。

### 4.2 查看日志确认启动成功

```bash
pm2 logs synctube --lines 30
```

应看到类似：

```
> SyncTube ready on http://0.0.0.0:3000
> Socket.IO attached on same port (path: /socket.io/)
```

按 `Ctrl+C` 退出日志查看（不会停进程）。

### 4.3 本机自测

```bash
curl -I http://127.0.0.1:3000
# 应返回 200 OK 或 302（首页重定向），HTTP/1.1
```

## 5. 设置开机自启

```bash
# 生成系统启动脚本（按提示执行 PM2 给出的最后一行命令）
pm2 startup systemd

# 提示类似：
#   sudo env PATH=$PATH:/usr/bin pm2 startup systemd -u root --hp /root
# 复制并执行那一行

# 保存当前进程列表，开机后自动恢复
pm2 save
```

验证：

```bash
pm2 list   # synctube 应在列表中
systemctl status pm2-root   # 应为 active (running)
```

## 6. 外部访问验证

在**本地浏览器**打开：

```
http://你的ECS公网IP:3000
```

应看到 SyncTube 首页。打开两个浏览器窗口，进同一房间，验证：

- 聊天消息实时同步
- 房主播放/暂停/拖进度条，成员跟着同步
- 房主离开后房主身份自动移交

如果**打不开**，按顺序排查：

1. ECS 安全组是否放行 3000/TCP（最常见原因）
2. `ufw status` 是否放行 3000
3. 服务器上 `pm2 logs synctube` 有无报错
4. 服务器上 `curl -I http://127.0.0.1:3000` 是否通
5. `ss -tlnp | grep 3000` 是否在监听 `0.0.0.0:3000`（不是 `127.0.0.1:3000`）

## 8. 常见问题

### Q1：访问很慢 / 首次加载久？
Next.js 生产模式首次访问会编译缓存，之后会快很多。如果持续慢，检查 ECS 带宽是否够（1Mbps 起步，多人观看建议 5Mbps+）。

### Q2：B站视频 iframe 加载不出来？
B站部分页面有 `X-Frame-Options` 限制，与服务器无关，是 B站本身策略。本地开发也会遇到。其他格式（mp4、YouTube 等）用 react-player 不受影响。

### Q3：多人同时看，房间状态会丢吗？
房间状态存内存，**PM2 重启 / 进程崩溃会清空所有房间**。这是当前设计取舍（项目文档明确"无数据库"）。如果稳定性要求高，后续可考虑加 Redis adapter 或落库。

### Q4：能不能用 80 端口而不是 3000？
可以，两种方式：

- 直接改 `ecosystem.config.cjs` 里 `PORT: 80`，然后用 `sudo pm2 restart synctube`（绑 80 需要 root）。
- 推荐做法：装 nginx 反代 80 → 3000，再加 Let's Encrypt 上 HTTPS（WebSocket 也走 wss）。后续要域名+HTTPS 时再升级。

### Q5：日志在哪？
`/opt/synctube/logs/out.log` 和 `error.log`（PM2 配置里指定的路径）。

## 9. 升级到 HTTPS（以后可选）

等你想用域名 + HTTPS 时：

1. 域名 A 记录指向 ECS 公网 IP。
2. `sudo apt install nginx`，配置反代 `80 → 127.0.0.1:3000`，并升级 WebSocket 头。
3. `sudo apt install certbot python3-certbot-nginx`，`sudo certbot --nginx -d 你的域名`。
4. PM2 端口保持 3000 不变，对外走 nginx 443。

## 附录：完整一键脚本（全新服务器从零部署）

> ⚠️ 在服务器上以 root 执行；先确保 3000 端口已在安全组放行；先把代码包 `synctube.tar.gz` 上传到 `/root/`。

```bash
#!/bin/bash
set -e

# 1. 系统更新 + 基础工具
apt update && apt upgrade -y
apt install -y curl git build-essential

# 2. Node.js 20 LTS
curl -fsSL https://deb.nodesource.com/setup_20.x | bash -
apt install -y nodejs

# 3. PM2
npm install -g pm2

# 4. 解压代码
mkdir -p /opt/synctube
tar -xzf /root/synctube.tar.gz -C /opt/synctube
cd /opt/synctube

# 5. 安装依赖 + 构建
npm install --omit=dev
npm run build

# 6. 启动 PM2
pm2 start ecosystem.config.cjs
pm2 save
pm2 startup systemd --service-name pm2-root --hp /root || true

# 7. 防火墙
ufw allow 22/tcp 2>/dev/null || true
ufw allow 3000/tcp 2>/dev/null || true

echo "=========================================="
echo "部署完成！访问 http://$(curl -s ifconfig.me):3000"
echo "=========================================="
```
