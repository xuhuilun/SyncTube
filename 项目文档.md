
# 项目提示词：异地同步观影网站（Next.js + Socket.IO）

## 角色与目标

您是一名全栈 Vibe‑Coding 工程师，需开发一个**异地同步观影网站**，主要支持B站，让不同用户在同一房间内同步观看在线视频，并附带实时聊天功能。项目需使用 **Next.js 15**（App Router）、**TypeScript**、**TailwindCSS**、**shadcn‑ui** 组件库、**Socket.IO** 实现实时通信。最终代码应可部署至 **Vercel**（需注意 WebSocket 限制，见部署备注）。

---

## 功能需求

### 1. 首页（`/`）
- **创建新房间**：点击按钮后生成一个随机 6 位字母数字房间 ID，并自动跳转至该房间页面。
- **加入已有房间**：提供一个输入框，用户输入房间 ID 后点击“加入”或按回车，跳转至对应房间页面。
- 若房间不存在，给出友好错误提示（Toast 或内联提示）。

### 2. 房间页面（`/room/[id]`）
- **视频播放器**：
  - 用户可输入任意在线视频 URL（支持B站、 HLS、MP4、YouTube 等常见格式，使用 `react-player` 或原生 `<video>`）。
  - 加载后显示播放器，并自动同步房间内所有用户的播放状态。
- **同步机制**（核心）：
  - 所有用户**播放/暂停**操作同步，由房主控制。
  - **拖拽进度条**（seek）同步，所有用户跳转到同一时间点。
  - 同步逻辑：由任一用户操作触发，通过 Socket 广播给房间内其他用户，接收方应用相同操作（需避免循环触发）。
  - 初次加入房间时，新用户自动同步当前播放状态（播放/暂停 + 当前时间）。
- **实时文字聊天**：
  - 显示在视频播放器旁或下方（响应式布局）。
  - 消息包含发送者昵称（用户可自定义昵称，或默认“匿名用户”+ 随机后缀）、发送时间、内容。
  - 消息列表自动滚动到底部。
- 语言聊天功能：
  - 房间内所有用户可以主动打开或关闭麦克风，在观影的同时互相交流
- **在线用户列表**：
  - 展示当前房间内所有用户的昵称。
  - 用户进入/离开时实时更新列表。

### 3. 数据存储
- 所有房间数据（房间成员、视频状态、聊天消息）**仅存储于内存**（Node.js 全局 Map 或对象）。
- 关闭网页或服务重启后数据清空，**无需数据库**。
- 每个房间的数据结构应包含：
  - `roomId`
  - `users`: `Map<socketId, { nickname, online: boolean }>`
  - `videoState`: `{ playing: boolean, currentTime: number, url?: string }`
  - `messages`: `ChatMessage[]`

## UI 与交互设计
- **主题**：暗色玻璃拟态（Glassmorphism），背景深色渐变，卡片半透明磨砂效果。
- **强调色**：柔和霓虹蓝（如 `#00d4ff` 或 `#6c63ff`），用于按钮、链接、高亮。
- **动画**：使用 `framer-motion` 添加页面切换、组件出现、消息淡入等过渡动画，提升流畅感。
- **响应式**：自适应手机（竖屏）和电脑（宽屏），布局灵活切换（聊天在手机下位于视频下方，桌面位于右侧）。

## 代码规范与架构
1. **所有交互页面组件**（如首页、房间页）必须标记 `'use client'`。
2. **业务逻辑拆分**：
   - 将 Socket 事件监听与发送封装到自定义 Hook（如 `useSocket`、`useRoom`），组件中仅调用 Hook 方法。
   - 视频同步逻辑单独封装成 Hook（`useVideoSync`），内部使用 Socket。
   - 聊天逻辑封装为 `useChat`。
3. **TypeScript 类型定义**（在 `types/` 目录下）：
   ```typescript
   interface Room {
     roomId: string;
     users: Map<string, OnlineUser>; // key: socketId
     videoState: VideoState;
     messages: ChatMessage[];
   }
   interface OnlineUser {
     socketId: string;
     nickname: string;
   }
   interface ChatMessage {
     id: string;
     sender: string;
     content: string;
     timestamp: number;
   }
   interface VideoState {
     url: string;
     playing: boolean;
     currentTime: number;
   }
   ```
4. **服务端 API**：
   - 使用 Next.js API Routes（`app/api/...`）处理房间创建、加入、获取房间信息等 REST 操作。
   - Socket.IO 服务器附加到 Next.js 服务（使用 `http.Server`），监听连接、断开、房间事件。
5. **不添加任何登录、注册、鉴权功能**，保持极简。

## 补充说明
- 房间内第一人加载视频后，其他人进入时自动同步当前状态。
- 聊天消息无需持久化，仅保留最近 200 条（防止内存溢出）。
- 若用户离开房间（关闭页面或断开连接），应从用户列表中移除，并通知其他成员。
- 确保错误处理（如无效 URL、Socket 连接失败）有友好提示。
